# Slot-Machine

Determined to make some kind of visual program not in console, I developed a slot machine that rolls numbers in Python.
For the first time, I was able to include animation into my code. Animation plays for:
- Placing a coin into the slot machine
- Pulling the handle
- Hitting a special jackpot

While small, this program was my first attempt at animating objects with code. I had to research how Python graphics function and everything about objects, text, etc. This was one of my projects not assigned by my highschool Computer Science class, so I had learned little to nothing about Python graphics. It marks a checkpoint in my journey to becoming a programmer.

At the time of me making this program, I was still set on disregarding the readability of my projects; I figured that only I would ever read my code and therefore would always know where things were. I hard-coded everything to be exact numbers, removing the possibility to change certain values without messing up others. It's very messy, and that's why it's important to me: looking over programs like these weeks later made me realize the value of the way programs are structured and made me rethink the methods I was using to code.

I believe the most important thing about this project was the impact it had on my mindset; it shifted my thought process from thinking about how I would view a program to how others might view my programs. One of the most essential parts of computer science is collaboration, and making sure your code is readable, changable, and coherent--even for yourself-- is vital for supporting a team. Thinking like this is not only generally good practice but also automatically makes you structure your programs while thinking about the future, saving time and energy. The creation of this program led me to this mindset.
